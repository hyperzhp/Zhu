\section{User Interface Design}

\subsection{Design Rationale}
The User Interface (UI) of the \textbf{Best Bike Paths (BBP)} application follows the \textbf{"Situational Design"} paradigm. Recognizing that cyclists interact with the app outdoors in Milan's busy streets, the design prioritizes:
\begin{itemize}
    \item \textbf{Context-Aware Visibility}: As seen in the Recording mode, the interface switches to a high-contrast dark theme to ensure legibility under direct sunlight.
    \item \textbf{Thumb Zone Navigation}: Critical actions (Start/Stop) are oversized and placed in the bottom 30\% of the screen for safe one-handed interaction.
    \item \textbf{Safety-First Alerts}: The system uses distinct color coding (Yellow/Red) to warn users of specific hazards like tram tracks.
\end{itemize}

\subsection{Screen Descriptions}
\begin{enumerate}
    \item \textbf{Dashboard}: As shown in \textbf{Figure \ref{fig:ui_flow} (A)}, the central hub automatically detects the user's location at \textit{Piazza Leonardo da Vinci} and suggests the safest route to \textit{Duomo}.

    \item \textbf{Active Recording}: During the ride (\textbf{Figure \ref{fig:ui_flow} (B)}), the UI minimizes distractions. It features a large speedometer and dynamic alerts. In this scenario, the app has detected upcoming \textbf{tram tracks} on \textit{Corso Venezia}.

    \item \textbf{Summary View}: Upon arrival (\textbf{Figure \ref{fig:ui_flow} (C)}), the user receives a "Safety Score" (94/100). The route visualization uses a gradient to indicate safe vs. caution zones.
\end{enumerate}
